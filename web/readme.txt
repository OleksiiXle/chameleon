В тестовом задании используется БД Chameleon (MySQL)
Таблицы:
1. config - для хранения
    пути к превью pathToPreview = "../preview/"
    имени временного файла превью tmpImage = "../preview/tmp/tmpimg.jpg"
2. authors - авторы книг
3. books - книги, (связь books.author_id =  authors.id)
4. filter - четыре поля - автор, название книги, дата с, дата по, всегда или одна запись или таблица пуста
    используется для наложения фильтра на выводимые из таблицы books записи, если фильтр пустой -
    выводятся все записи, если нет - выводятся записи в соответствии с фильтром
5. user - список авторизованных пользователей

Контроллеры:
1. SiteController.php - регистрация нового пользователя Signup, LOGIN, LOGOUT.
2. BooksController.php:
    - actionIndex() - если пользователь не зарегистрировался и не вошел под своей учетной записью - возврат,
    рендерит основной вид index, передает ему модель с данными фильтра $queryModel, модель с данными книг соответственно фильтру
    $searchModel и провайдера $dataProvider;
Вид index:
- кнопка перехода к форме создания новой записи
- форма для изменения фильтра с кнопками "применить фильтр" и "удалить фильтр"
- GridView для вывода информации о книгах соответственно установленному фильтру:
    при клике на Превью - увеличение его размера
    сортировка, пагинация
    кнопка просмотр - модальное окно
    кнопка редактировать - отдельное окно редактирования  с возвратом в исходное состояние
        index с учетом сортировки, пагинации и фильтра, для запоминания исходного состояния используется сессия
    кнопка удалить запись
Вид _form:
используется для редактирования имеющейся или создания новой записи
- возможность предпросмотра превью с помощью ajax (выбрать файл, обновить превью)
- по умолчанию файл превью - tmpImage = "../preview/tmp/tmpimg.jpg", если пользователь не выбрал другой файл,
превью книги формируется из этого файла,
- при изменении превью в "newPreviewTxt" записывается имя временного файла, если оно не пустое,
  то впоследствии временный файл записываеться в определенную папку, новое имя -
  'preview_' + ID + расширение, для файла иконки превью вначале добавляется "i".

 После редактирования или добавления вызывается вид view для предварительного просмотра того что получилось, можно
 опять редактировать запись, можно удалить



При добавлении файлы изображения масштабируются - превью - 2000, иконка - 500
в GridView выводится иконка, во все остальные виды - само превью.
